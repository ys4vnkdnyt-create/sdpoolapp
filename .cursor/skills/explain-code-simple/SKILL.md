# Explain code simply

## When to use

- User asks what a file, function, or flow does
- User wants a walkthrough before changing code
- User says they are learning and want less jargon

## How to respond

1. Start with **one sentence**: what this code is for.
2. Then explain in **short bullets** (one idea each).
3. Show the **TypeScript** being discussed (cite the real file when possible).
4. Define any tech word the first time it appears (plain English).
5. End with **one optional next step** the user can choose (do not dump a long lesson).

## Do not

- Do not rewrite large amounts of code unless the user asks
- Do not assume advanced backend knowledge
- Do not paste huge dumps; stay bite-size unless they ask for more
